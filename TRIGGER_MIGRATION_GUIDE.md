# 🔥 Calendary - 트리거 방식으로 마이그레이션 완료!

## ✅ 변경 사항

### **이전 방식 (서버 엔드포인트)**
```
회원가입 → 서버 API 호출 → 카테고리 3개 생성 → metadata 플래그 설정
```
- 복잡한 로직 (중복 체크, RLS 우회, 에러 처리)
- 프론트/서버 코드 필요
- metadata 플래그로 중복 방지

### **새로운 방식 (DB 트리거)**
```
회원가입 → DB 트리거 자동 실행 → 카테고리 3개 생성
```
- **완전 자동화** (코드 필요 없음)
- **DB 레벨 보장** (절대 빠뜨릴 일 없음)
- **빠름** (네트워크 요청 없음)

---

## 🚀 적용 방법

### 1️⃣ Supabase에서 트리거 설치

1. **Supabase Dashboard** 접속
2. **SQL Editor** 메뉴 이동
3. `/TRIGGER_SETUP.sql` 파일 내용 복사
4. **RUN** 버튼 클릭 ✅

---

### 2️⃣ 기존 중복 카테고리 정리 (선택사항)

현재 DB에 중복 카테고리가 있다면:

1. **SQL Editor**에서 `/CLEANUP_DUPLICATES.sql` 파일 내용 복사
2. **RUN** 버튼 클릭 ✅
3. 결과 확인: 각 유저당 최대 3개만 남음

---

### 3️⃣ 테스트

#### **신규 회원가입 테스트**
1. 새 계정으로 회원가입 (이메일 또는 구글)
2. 로그인 후 카테고리 확인
3. **기대 결과**: 자동으로 3개 생성 (개인, 업무, 휴가)

#### **기존 사용자 테스트**
1. 기존 계정으로 로그인
2. 카테고리 확인
3. **기대 결과**: 기존 카테고리 유지 (중복 생성 없음)

## 🐛 트러블슈팅

### Q1. 트리거가 실행되지 않아요
**A.** Supabase SQL Editor에서 실행했는지 확인하세요. 로컬 파일에서는 실행되지 않습니다.

### Q2. 기존 사용자도 카테고리가 생성되나요?
**A.** 아니요. 트리거는 `auth.users` 테이블에 **새로 INSERT될 때만** 실행됩니다. 기존 사용자는 영향받지 않습니다.

### Q3. 중복 카테고리가 여전히 있어요
**A.** `/CLEANUP_DUPLICATES.sql`을 실행하여 기존 중복을 정리하세요.

### Q4. 트리거를 삭제하려면?
```sql
DROP TRIGGER IF EXISTS trigger_create_default_categories ON auth.users;
DROP FUNCTION IF EXISTS public.create_default_categories_for_new_user();
```
